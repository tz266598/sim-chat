# SIM-Chat 手动启动指南

## 🚀 方法一：一键启动（推荐）

### Windows 系统
直接双击运行项目根目录下的：
- **启动**: `start.bat`
- **停止**: `stop.bat`

### Linux/Mac 系统
```bash
# 赋予执行权限
chmod +x start.sh stop.sh

# 启动项目
./start.sh

# 停止项目
./stop.sh
```

## 🛠️ 方法二：手动启动

### 步骤 1: 启动后端服务

打开命令行，进入后端目录并启动服务：

```bash
cd C:\Users\26659\Desktop\sim-chat\backend
python -m uvicorn app.main:app --host 0.0.0.0 --port 8001
```

**说明**：
- `--host 0.0.0.0`: 允许外部访问
- `--port 8001`: 指定端口为8001
- 窗口不要关闭，保持运行

### 步骤 2: 启动前端服务

打开**新的**命令行窗口，进入前端目录并启动：

```bash
cd C:\Users\26659\Desktop\sim-chat\frontend
streamlit run app.py --server.port 8501
```

**说明**：
- `--server.port 8501`: 指定端口为8501
- 窗口不要关闭，保持运行
- 首次启动会自动打开浏览器

## ⚙️ 启动前检查

### 1. 检查Python环境
```bash
python --version
# 需要 Python 3.8+
```

### 2. 检查依赖安装
```bash
# 后端依赖
cd backend
pip install -r requirements.txt

# 前端依赖
cd frontend
pip install -r requirements.txt
```

### 3. 检查配置文件
确保 `.env` 文件存在且配置正确：
```bash
MINIMAX_API_KEY=your_api_key_here
MINIMAX_GROUP_ID=your_group_id_here
```

## 🛑 停止服务

### 方法一：使用停止脚本
双击运行 `stop.bat`

### 方法二：手动停止
在运行后端和前端的命令行窗口中按 `Ctrl + C`

### 方法三：强制停止（Windows）
```bash
taskkill /F /IM python.exe /T
```

## 🔧 常见问题

### Q1: 端口被占用
**错误**: `error while attempting to bind on address ('0.0.0.0', 8001)`

**解决**: 
```bash
# Windows: 查找并停止占用端口的进程
netstat -ano | findstr :8001
taskkill /F /PID <进程ID>
```

### Q2: 依赖缺失
**错误**: `ModuleNotFoundError: No module named 'xxx'`

**解决**:
```bash
# 重新安装依赖
cd backend
pip install -r requirements.txt

cd ../frontend
pip install -r requirements.txt
```

### Q3: API密钥未配置
**错误**: `MINIMAX_API_KEY` 相关错误

**解决**:
1. 复制配置文件: `copy .env.example .env`
2. 编辑 `.env` 文件，填入真实的API密钥

### Q4: 向量数据库路径错误
**错误**: `Collection not found`

**解决**:
确保数据库目录存在：
```bash
# 检查路径
C:\Users\26659\Desktop\sim-chat\data_processing\chroma_db
```
